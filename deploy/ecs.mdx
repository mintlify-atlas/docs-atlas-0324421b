---
title: "Deploy to Amazon ECS with Terraform"
description: "Provision an ECS Fargate cluster with all retail store services and managed AWS backend dependencies using the included Terraform module."
---

The ECS Terraform module deploys all retail store services as ECS Fargate tasks, with ECS Service Connect handling inter-service communication. Application dependencies are provisioned as AWS managed services, and observability is configurable via Container Insights and OpenTelemetry.

<Warning>
  This module creates AWS resources that will incur costs in your account. Review the resources being created before proceeding, and destroy them when you are done.
</Warning>

## What's created

| Resource | Details |
| --- | --- |
| VPC | Public and private subnets |
| ECS cluster | Fargate compute for all application services |
| ECS Service Connect | Handles traffic between application services |
| Amazon RDS | Relational database for the catalog and orders services |
| Amazon DynamoDB | Table for the cart service |
| Amazon ElastiCache | Cache layer for the application |
| Container Insights (optional) | Enhanced monitoring for ECS cluster metrics |
| OpenTelemetry (optional) | AWS Distro for OpenTelemetry with trace collection |

## Prerequisites

- AWS CLI and Terraform installed locally
- AWS CLI configured and authenticated with the account you want to deploy to

## Deploy

<Steps>
  <Step title="Navigate to the Terraform directory">
    ```bash
    cd terraform/ecs/default
    ```
  </Step>
  <Step title="Initialize Terraform">
    ```bash
    terraform init
    ```
  </Step>
  <Step title="Review the plan">
    ```bash
    terraform plan
    ```
  </Step>
  <Step title="Apply the configuration">
    ```bash
    terraform apply
    ```

    Terraform will prompt for confirmation before creating resources. The process takes approximately 15 minutes.
  </Step>
  <Step title="Access the application">
    Retrieve the application URL from Terraform outputs:

    ```bash
    terraform output -raw application_url
    ```

    Enter the URL in your browser to access the application.
  </Step>
</Steps>

## Variables

| Name | Description | Type | Default | Required |
| --- | --- | --- | --- | :---: |
| `environment_name` | Name of the environment used for all created resources | `string` | `retail-store-ecs` | yes |
| `opentelemetry_enabled` | Installs AWS Distro for OpenTelemetry and configures trace collection | `bool` | `false` | no |
| `container_insights_setting` | Container Insights setting for the ECS cluster. Must be `enhanced` or `disabled`. | `string` | `enhanced` | no |
| `lifecycle_events_enabled` | Sends ECS lifecycle events to CloudWatch Logs for Container Insights dashboards. Only available when `container_insights_setting` is `enhanced`. | `bool` | `false` | no |

## Outputs

| Name | Description |
| --- | --- |
| `application_url` | URL where the application can be accessed |
